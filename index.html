Agent Based Model (ABM) and Multi-Agent System:

So, what is Multi-Agent System? To begin, let’s first talk about what is an agent.

If one ever learned Java, or C ++, will know of a idea called Object Oriented Programming. An object can be an entity which contains some attribute and have some functions. This is very close to the idea of an agent. Jacques Ferber first defined that an agent can be a physical or virtual entity that can act, perceive its environment (in a partial way) and communicate with others, is autonomous and has skills to achieve its goals and tendencies.1 Below is a representation of an agent in a simulated environment. The agent is the blue dot (let's call it BLUE, I love cute names)  who only live if it's eats a green dot in a limited amount of time. As you can see, BLUE can perceive things (link with a highlighted beam) that are reasonably close to it. And the main goal for BLUE is to live and not die (as pretty much everyone is). BLUE can decide which grass to eat or just not eat and wondered around. Yes, the smell of freedom.

As you can tell, an agent is not so fun, seeing as it's just doing a thing that it suppose to do and not many variation is found. Same in real life, most stuff can not be accomplish by one person. Some tasks requires  to coordinate people, and in this case agents to fulfill.

As you can tell, this is an extremely loose definition. The looseness came from that an agent can be instantiate in a lot of situations. // A common use would be digital society where each agent is considered an individual who are of some characteristics and interact with other individual as how they are defined. These agents can also work as a proxy of a party and negotiate with other agent in order to make a deal. The list goes on a on. // A multi-agent system put multiple agents in a predefined environment. The environment can be dynamic or static depends on the usage. The agent can also be connected in a specific topology.
